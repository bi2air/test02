## DCI-323 · Debug cell 118-A02 (CSV3 production score failure) — forensic kit kickoff

**Why.** Cell **118-A02** in the CSV4 pipeline (the CSV3-derived raw-score input feeding `csv4_score_prediction` 118-B01) crashed Sat Apr 11 13:11 UTC with `ValueError: bins must be monotonically increasing or decreasing` at `csv3-prod/lib_csv3/pipeline/score_csv3/base.py:308` in `transform_piecewise_linear()`. The CSV3 score-conversion step is the upstream of CSV4 scoring; failure here cascades the whole CSV4 weekly run. Need a forensic kit that's read-only (so it can run against archived data without risk of corrupting prod), independent (failure of one script doesn't gate others), and standalone-friendly (some pure-Python, some PySpark) so partial diagnosis is possible without a full Spark cluster.

> ### 🛠 What I did
> Stood up the `csv3_digitize_debug_kit/` (16:04–16:13) — six initial files, all read-only and independent:
> - **`1_check_logs.sh`** (114L) — grep Airflow driver log for the `print(transform_dict)` capture (fastest, free, surfaces the root-cause array directly).
> - **`2_inspect_serialized_models.py`** (119L) — load `.pkl` model artifacts, verify structure + coef/bias values (pure Python, no Spark).
> - **`3_data_diagnostics.py`** (177L) — PySpark — read actual data, compute distributions and knot positions on the failing date.
> - **`4_reproduce_failure.py`** (228L) — synthetic harness: which logodd-distribution drift breaks the transform? (pure Python, no Spark or data needed; runs anywhere).
> - **`README.md`** + **`TROUBLESHOOTING.md`** — quick-start in-order runbook + decision tree across 4 scenarios (drift / tied percentiles / fragile calibration / misconfiguration), each with a concrete remediation path.

**Result.** Forensic kit ready. The two stacked bugs identified by the kit: (1) **format mismatch** — caller passed `transform_dict` (a `{'x':[…],'y':[…]}` dict) where `dict_pct` (a plain quantile-position list) was expected; iterating a dict yields keys → Spark sees `array(x, y)` as column refs → `AnalysisException`. (2) **distribution drift** — even with correct format, `csv3pdl_raw` P13.76 = 499.85 fell below the hardcoded prefix ceiling of 500 → non-monotonic x-knots. Apr 14–15 work continues with recalibration scripts 5–7.
