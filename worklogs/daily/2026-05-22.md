## What the OKRs target

**O2-KR2** wants production call-code QC to catch regressions automatically so human bug-reports drop ≥75%. **O4-KR1** wants the data-collection → evaluation loop to run as scheduled infrastructure, not hand-run commands. The bottleneck both target: today there's no automated daily check that re-scores production calls and flags where the classifier disagrees with itself across models — drift is only found when a human files a bug. Today builds the measurement engine that closes that gap.

## What we did

Completed the **postcall auto-QC pipeline prototype** — a daily Cloud Run Job that pulls yesterday-NY's Affirm `call_reports` rows from BigQuery (stratified per `(vendor_id, call_code, direction)`), reruns each conversation through the postcall classifier under **three voters sharing one byte-identical prerendered prompt** (V1 = production yaml's model resolved as-of the call date via `git log --first-parent --before` on `agi-sm-dsci-configs`, V2 = `gemini-2.5-pro`, V3 = `gpt-5`), and writes per-call agreement booleans to BigQuery. Delivered as **8 modules under `src/postcall/auto_qc/`** (sampler, csv-adapter, yaml-asof resolver, 3-voter fan-out, BQ writer, daily orchestrator with resume cursor), a `deploy/auto_qc/` Docker bundle, a local cycle wrapper, and a ~16KB end-to-end deploy README. Pilot scope = Affirm only (`vendor_id=16`).
