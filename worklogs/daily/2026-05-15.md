## Approach

1. Start from a small, inspectable slice before scaling a workflow.
2. Preserve each run as a baseline so the next pass can be compared cleanly.
3. Add one constraint per iteration: confidence scoring first, phase alignment second, then review readiness.
4. Separate structural cases from user-utterance evidence so the model is not blamed for missing evidence that does not exist.
5. Move monitoring from analysis output to operating loop only after it has clear ownership, report surfaces, and no-noise rules.
6. Convert repeated maintenance patterns into skills once the pattern is stable enough to reuse.
