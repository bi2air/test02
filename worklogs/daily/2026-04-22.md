## Standardized reporting + frozen testset for postcall classifier

**Why.** AGI-7771 was scheduled for delivery to UAT Apr 23. Two of the existing pieces — report shapes and testset code — were drifting across ad-hoc scripts. Without standardization, the AGI-7771 result would be a one-off (manually defensible but not reproducible by anyone else), and the AGI-9466 follow-on (11 ACs) would need a fresh harness per ticket. Today's infra is the difference between "this delivery worked once" and "this delivery is the new baseline".

> ### 🛠 What I did
> Shipped four reusable infrastructure pieces:
> 1. **`build_reports.py`** — single producer of two output shapes:
>    - `<run>_full.csv` — postcall-output: every `CallCode` row + `extra_info` columns containing `extracted_info.{list_call_codes, callcode_reason, summary}` plus compliance/scorecard fields.
>    - `<run>_debug.csv` — predicted-vs-expected join per `(conversation_id, tc_id)` with column names that make spreadsheet diff viewers auto-render correctly.
> 2. **Frozen testset** at `testing/inputs/testset_agi7771_v1.csv` — 17 rows × 15 cols, sha256 `d55a5796`. Produced once via `backtest_from_xlsx.py --xlsx <path>`; consumed thereafter via `--testset-csv <path>` to skip the xlsx + Datadog re-join entirely.
> 3. **`rerun_postcall.py`** — diagnostic-only classifier runner (not a backtest). Feeds frozen testset through the classifier; writes `manifest.json` + a `data/examples/<conv_id>/` bundle per case (`postcall_result.json`, `run_meta.json`, `reflection_context_*.json`, `llm_call_*.json`, `prompt_detect_call_code_prerendered.txt`, `postprocess_eval.json`). Useful for "what is the classifier doing on row N?" inspection without invoking the full verdict layer.
> 4. **Canonical debug-artifact layout** at `data/examples/<conv_id>/` — mirrors the production debug bundle shape. Documented the `rsync --exclude=*.txt` gotcha that silently drops the prerendered prompt files (which look like noise but are the only on-disk record of what the LLM actually saw); explicit include list now used for every zero-server sync.
>
> Plus 3 DCI tickets created to scope multi-day work: telco score maintenance (DCI-321), sub-population selection (DCI-322 — closed same day), MBF schema check (DCI-328 — closed same day).

**Result.** The Apr 23 AGI-7771 delivery ran end-to-end on this infrastructure — reproducible by anyone via `backtest_from_xlsx.py --testset-csv testing/inputs/testset_agi7771_v1.csv`. The same harness is reused for the AGI-9466 11-AC evaluation per `docs/AGI9466_PIPELINE.md`, saving a separate eval pipeline per AC.
