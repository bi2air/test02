## AGI-8632 · Compliance review UI + R4 testset (120 rows, 3-run consensus)

**Why.** Continues the **AGI-8632** ([C09] Call disposition codes audited & redial logic tested) audit line started Apr 14. The Apr 14 baseline (63.5%, with WCR labels mostly wrong) showed labels themselves were uncertain — not just the classifier. Today's work splits *label quality* from *classifier quality*: build a re-labeled gold testset using 3-run consensus + a Flask UI for human review of disagreements. Without this split, AGI-8632 conclusions would conflate two different failure modes.

> ### 🛠 What I did
> - **Flask `/compliance` UI** — web tool to review classifier outputs vs labels, mark disagreements, capture relabel decisions.
> - **R4 testset, 120 rows** — built via `build_testset_r4.py`. Larger and more stratified than the Apr 14 testset (which surfaced the under-bar codes).
> - **3-run consensus labeling** — each row gets 3 independent classifier runs; consensus = same code 2/3+ times. Disagreements go to the UI for human adjudication. This is the **"Shell Splice"** pattern: split low-confidence rows from high-confidence, only re-label the low.
> - **Relabel pipeline** — produced `compliance_testset_0414_relabeled.csv` (R1 cohort) and `compliance_testset_r2_0414_relabeled.csv` (R2 cohort) as the new gold standard.

**Result.** R4 testset (120 rows) ready as the new compliance-class gold; label quality decoupled from classifier quality so future runs can attribute mis-classifications correctly.

## DCI-322 · 10k-SID stratified sampler — design spec

**Why.** CSV3 pipeline is migrating to cloud. Before cutover, a representative 10k-SID sample is needed to run scoring on both on-prem and cloud, then byte-compare outputs. Need to spec the sampling strategy: how many primaries, how to stratify, how to ensure sample-internal score-fidelity (i.e. graph features still resolve correctly inside the sample).

> ### 🛠 What I did
> Drafted **`stratify_sampling.md`** (15:46) — design doc for the two-tier sampler:
> - **Tier 0 — ~250 primaries** stratified on a 4×4 (age-bin × csv3_score-bin) matrix.
> - **Tier 1 — ~9,750 1st-hop neighbours** (9-month graph lookback) of the primaries.
> - **Score-fidelity guarantees**: primaries get **EXACT** scores (1st-hop is in the sample, all graph features resolve); 1st-hop gets **~99%** (only neighbour-score feature degrades, since 2nd-hop is missing).
> - Activity-window filter: each primary must have `activity_4w=1` at ≥3 of 5 checkpoints across the May–Dec 2025 window — guarantees scoreability against the 33-week clean-data horizon (per the Apr 8 audit).

**Result.** Spec on file; first end-to-end stats run happens Thu Apr 16; production code (`sample_10k_sids.py` 706L + `read_with_archive.py` 73L) ships Fri Apr 17.
